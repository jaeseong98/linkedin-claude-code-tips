# 구현 Agent와 테스트 Agent를 분리해 컨텍스트 격리하기

Claude Code에서 서브에이전트(Agent 도구)를 활용할 때, 기능 구현을 담당하는 에이전트와 테스트를 작성하는 에이전트를 별도 컨텍스트로 분리한다. 같은 컨텍스트를 공유하면 구현체에 맞춰 테스트를 역설계하거나 mock으로 속이는 경향이 있으므로, 격리된 에이전트가 명세(spec)만 보고 독립적으로 테스트를 작성하게 한다.

---

## 사용법

지금 구현된 [기능명]에 대한 테스트를 작성해줘. 단, 구현 코드를 보지 말고 아래 명세(spec)만 참고해서 작성해줘. mock은 외부 I/O에만 허용하고, 핵심 로직은 실제 값으로 검증해야 해. 명세: [명세 내용]
